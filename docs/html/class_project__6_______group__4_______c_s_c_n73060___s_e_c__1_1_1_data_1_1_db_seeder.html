Detailed Description

Seeds the database from SQL files.

The seeder runs at application startup when the database is empty. Splits the SQL into batches and applies them inside transactions to avoid large single-statement execution.

Seed the database.

This method is idempotent: it will skip seeding when CPUs already exist. It logs progress and rethrows unexpected exceptions after logging for the caller to handle.
